Architecture

A fleet of specialized agents that crawl, analyze, build, migrate, configure, and validate — transforming any website into AEM Edge Delivery Services.

● Code → GitHub
● Content → da.live

Agent Orchestration

The Orchestrator owns the session and gates every phase. Workers transform; Tier 1 scripts validate automatically; Tier 2 verifier agents apply judgment only where scripts can't. Retries loop until pass or escalation.

Pipeline

Seven phases, top to bottom. Each phase runs a worker agent, validates with Tier 1 scripts, then optionally with a Tier 2 verifier agent. Each worker also writes phase documentation. Autonomous fallback at every step — human escalation only as last resort.

Agent Roster

Every agent in the system — workers that do the job, and verifiers that check the work. Workers never self-review. Verifiers are separate agents with different prompts and explicit acceptance criteria. Not every phase needs a Tier 2 verifier — Config and Integration QA are fully validated by scripts.

Two-Tiered Verification

Every phase output passes through verification. Tier 1 catches structural issues instantly (including visual diffs via pixelmatch). Tier 2 catches quality issues with judgment. Retries pass only the latest feedback to prevent context window exhaustion.

Tier 1 — Scripts

Deterministic

Runs after every worker step. No LLM. Near-zero cost. Catches ~80% of issues.
  • JSON / YAML schema validation
  • File existence & structure checks
  • Linting (npm run lint)
  • Cross-referencing IDs & counts
  • Visual diff scoring (pixelmatch)
  • HTTP reachability checks
Tier 2 — Verifier Agents

Judgment-Based

Separate Claude agent per check. One LLM session. Used only where reasoning is needed.
  • "Does this block palette make sense?"
  • "Is this content model author-friendly?"
  • "Is content complete, not garbled?"
  • "Do blocks decorate correctly?"
  • Visual diffs in ambiguous range (0.7–0.9)

Output Delivery

The migration produces five deliverables. Together, the GitHub repo and da.live content form a working EDS site previewable immediately.

Code

GitHub Repository

Blocks, styles, scripts, and configuration. Connected to da.live via fstab.yaml.
  • blocks/ styles/ scripts/
  • head.html fstab.yaml
  • helix-query.yaml robots.txt
Content

da.live Site

HTML pages and media uploaded via Source API. Immediately previewable at aem.page.
  • index.html nav.html footer.html
  • products/widget.html …
  • metadata.json redirects.json
Documentation

Migration Docs

Per-phase reports, authoring guide, and a final MIGRATION-REPORT.md compiled by the Orchestrator. Pushed to GitHub.
  • docs/MIGRATION-REPORT.md
  • docs/phase1–6 reports
  • docs/authoring-guide.md
Quality

QA Report

Per-page pass/fail with Lighthouse scores, visual diff scores, preview links, and da.live edit URLs.
Audit

Migration Log

Every verifier verdict, regression, autonomous fallback decision, and cost checkpoint. Full traceability.
